# 移动端通用测试经验库 (Universal Test Heuristics)
# 版本: v1.1 | 适用对象: Android/iOS 自动化 Agent

---

## 1. 交互与 UI 规范 [Interaction & UI]
- **幂等性自检 (Idempotency)**: 
    - 描述: 连续快速点击同一个按钮（如：提交、开关、进入详情页）。
    - 预期: 系统应处理防抖（Debounce），避免产生重复请求、打开多个重复页面或逻辑错乱。
- **中断处理 (Interruption)**: 
    - 描述: 操作进行中突然有电话拨入、闹钟响铃、切换至后台或触发系统分屏。
    - 预期: 返回后应用状态应保持一致，不应出现数据丢失或进程崩溃。
- **视觉反馈 (Visual Feedback)**: 
    - 描述: 点击后的置灰态、Loading 动画、或是操作成功/失败的 Toast/SnackBar 提示。
    - 预期: 必须有明确的视觉反馈告知用户当前状态，禁止点击后无响应。
- **极端文本 (Extreme Text)**: 
    - 描述: 长字符串（如超长 SSID、多国语言用户名）。
    - 预期: UI 不应出现重叠（Overlapping）或被生硬截断，应支持省略号显示或自动换行。

---

## 2. 网络专项 [Network Engineering]
- **状态互斥 (State Mutex)**: 
    - 描述: 开启飞行模式时，检查 Wi-Fi 和数据网络的状态。
    - 预期: Wi-Fi 和数据网络必须自动关闭或进入“不可用”灰色态。
- **弱网环境 (Weak Network)**: 
    - 描述: 在 2G/3G 或高丢包率环境下执行接口请求。
    - 预期: 接口超时（Timeout）后有合理的降级处理，支持手动重试，禁止长时间白屏。
- **网络切换 (Network Roaming)**: 
    - 描述: 从 Wi-Fi 切换到移动数据，或从 5G 切换到无网环境。
    - 预期: 应用需具备静默重连（Seamless Reconnect）能力，关键操作不应报错中断。
- **离线拦截 (Offline Interception)**: 
    - 描述: 在断网状态下启动功能或执行操作。
    - 预期: 应预判并拦截操作，弹出网络异常提醒，防止产生无效的底层调用。

---

## 3. 异常边界与安全 [Edge Cases & Security]
- **权限拦截 (Permission Handling)**: 
    - 描述: 用户在系统弹窗中拒绝必要权限（如定位、存储）。
    - 预期: 应用不应崩溃，且需弹出二次引导，告知用户该功能为何需要此权限。
- **电量/存储极端情况 (Resource Constraints)**: 
    - 描述: 电池电量极低（<15%）或手机存储空间几乎已满。
    - 预期: 高功耗操作（如同步、下载）应有预警或自动暂停，防止写文件失败导致崩溃。
- **多设备登录 (Session Conflict)**: 
    - 描述: 同一账号在另一台设备登录导致当前设备被踢下线。
    - 预期: 页面应立即强制跳转至登录页或显示“会话过期”提示。

---

## 4. 逻辑一致性 [Logic Consistency]
- **闭环路径 (Path Closure)**: 
    - 描述: 凡是 PRD 描述了“如何开启”的功能。
    - 预期: 必须考虑“如何关闭”以及关闭后相关资源的释放（如后台扫描是否停止）。
- **前置依赖 (Prerequisites)**: 
    - 描述: 执行 A 操作前依赖 B 条件。
    - 预期: 如果 B 条件不满足，A 按钮应处于隐藏、置灰态或点击后跳转至 B 的配置页。
